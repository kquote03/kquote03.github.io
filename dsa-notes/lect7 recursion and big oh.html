1. Find minimum distance

Let’s say we have an unordered array of size 0 to n-1 full of numbers, we want to find the smallest difference between all the numbers.

Our strategy would be to loop over the whole array for each element and storing the smallest difference. 

static void main(String[] args){
System.out.print(findMiniDif({1,2,3,4,5,80}));
}
public static int findMiniDif(int[] A){
    int minimum = Integer.MAX_VALUE;
    //A.length -1 because we dont need to subtract the last item
    for(int i = 0; i<A.length - 1; i++){
        for(int j = i+1; j<A.length; j++){
            if(minimum > Math.abs(A[i] - A[j]))
                minimum = Math.abs(A[i]-A[j]);
        }
    }

    return minimum;
}

Complexity: O(n) * O(n) = O(n2)

However, this is very inefficient, but we can take a different approach, we can sort the array.

Always remember that sorting can sometimes actually improve performance. This is because sorting allows to use a better more efficient algorithm. 

public static int findMinSorted(int[] A){
    int min = Integer.MAX_VALUE;

    sort(A); //Imaginary method to sort A

    for(int i = 1; i<A.length=1; i+1){
        if(Math.abs(A[i]-A[i+1]) < min)
            min = Math.abs(A[i]-A[i+1]);
    }
    return min;
}

3. Binary Search

Very important 

//This is just a wrapping function, ignore it for now.
public static int binarySearch(int[] A, int k){
    return bs(A,0,A.length-1, k);
}

//-1 if the item doesn't exist
//else returns the position of the item in the array
public static int bs(int[] A, int f, int l, int k){
    if(l < f) return -1; //does not exist by default
    int mid = (f+l)/2;

    if(A[mid] == k) return k;

    if(k > A[mid]) return bs(A, mid+1, l, k);
    else return bs(A, f, mid-1, k);
}

Complexity: O(log n) because we keep dividing by 2
