1. List ADT

  • Maintains order
  • Very flexible
  • Allows repetition
  • Items in list have position (may or may not be important)
  • Items can be added anywhere
  • The index of a list starts with 1, not 0. Unlike the array.

1.1. Interface

1.1.1. add(newEntry) : void

Adds to the end of the list

1.1.2. add(newPosition, newEntry)

Adds at a specific position, if something was already in that position, say position 5, then whatever was already at position to 5 will be at 6 (along with the rest of the list being pushed down) and the new item takes it place

1.1.3. remove(givenPosition) : <T>

Removes the item at that position and returns the item

1.1.4. clear() : void

just removes all the items in the list and throws it in the trash

1.1.5. replace(givenPosition, newEntry) : <T>

Replaces the item at that position with a new item, the replaced entry is returned

1.1.6. getEntry(givenPosition) : <T>

Just retrieves the item at the specified position

1.1.7. getLength() : int

Retrieves the length of the list (how many items)

1.1.8. toArray() : <T>[]

1.1.9. contains(anEntry) : boolean

Checks if an item exists in the list

1.1.10. isEmpty() : boolean

Checks if the list is empty

1.2. Array or LinkedChain

Array, because accessing elements will be O(1) but in LinkedChain it could be O(n)

1.3. Something the bag has but the list doesn’t

The getFrequency() method.

2. Finding max grades in a list

static int findMax(ListInterface<Integer> L){
    int max = Integer.MIN_VALUE;
    for(int i = 1; i <= L.getLength(); i++){
        if(L.getEntry(i) > max)
            max = L.getEntry(i);
    }
    return max;
}
