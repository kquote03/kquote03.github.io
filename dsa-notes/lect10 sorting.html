1. Merge sort

  • Uses “Divide and Conquer” paradigm
  • The algorithm is divided into two parts
  • Used to merge two already sorted arrays into one big one.

1.1. Merging two sorted arrays into one big one

  • Index of first array let’s say is n
  • For the second it’s m
  • First of all we compare the first two items
  • We copy the smaller item, let’s say m had the smaller item, so now we increment m by 1
  • We don’t increment n we continue comparing until an item in m is larger than n, only then can we increment n

2. Quicksort

  • We pick one element called the pivot, any element
    • If every element to the left is less than the first pivot and all the elements to right are greater, then the pivot has reached the final destination
  • Find the position of the pivot in the final sorted array
  • Then divide the array like in merge sort (keep dividing)
  • Find the pivot of the newly divided array
  • Keep dividing and finding the pivot
  • Merge the arrays

2.1. Complexity

  • Worst case: \(O(n^2)\) (when our pivot is always coming at the end)
  • Average case: \(O(n \log n)\)
