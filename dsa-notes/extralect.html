1. For nested for loops

  • We don’t care about starting conditions
  • We don’t care about the increment as long as it is linear (+ or -)
  • We mainly care about the stop condition

2. Extra tips

  • We don’t care about small numbers for big-Oh. We care about very larger numbers, remember than big-Oh should go for infinity.
    • In other words, we have to look at the algorithm when n has no bounds.
  • Not that important but
    • \(\log_2(n) = \log_3(n) = \log_{\text{whatever}}(n)\)

3. Notation (Mainly important for DDA)

  • \(O\) worst case (We will never be slower)
  • \(\Omega\) best case (We will never be faster)
  • If \(O\) and \(\Omega\) are the same, then it’s \(\theta\)

4. Question

foo1(int n){
    for (int i = 1; i<n; n++ )
        for(int j = 1; j<n; j++)
}

foo2(int n){
    for(int i =0; i<n; i++)
        for(int j = 1; j<n; j = j*2)
}

foo1 took 1 hour for 1024, how long will it take for input of 512?

First off find complexity → \(O(n^2)\)

1024 60 mins 1024 * 1024 iterations (1M)
512 15mins ; 60*0.25 → 512*512 iterations (0.25M)

foo2, complexity is \(n\log(n)\)

1024   \(1024*\log_2(1024) = 10240\)
     

Basically we have to find the number of operations per second, then find the number of operations for the second algorithm, and to find the time just multiply it by the number of ops/s
