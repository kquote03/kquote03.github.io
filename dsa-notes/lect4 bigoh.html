1. Lower polynomials don’t matter

for \(a_kn^k + a_{k-1}n^{k-1} + ... + an + 1\) then we immediately look at the highest power and \(O(n^k)\) immediately take it.

2. another rule

if we have \(f_1(n) + f_2(n) + f_3(n)\) then \(O(max(f_1, f_2, f_3))\)

3. General guidelines

3.1. If the number of instructions doesn’t depend on the input

It is usually \(O(1)\), since there are a constant number of commands.

3.2. If we have nested loops we do multiplication

For example 

for(int i =0; i<n; i++)
    for(int j = 2; 1<n; j+=5)

Here we have \(O(n) * O(n)\) which results in \(O(n^2)\)

3.3. If the increase / decrease in a loop is * or / then O(log(n))

For example

for(int i = 1; i<=n; i=i*i) is \(O(log(n))\)

for(int i = 1; i<=n; i=i/2) is \(O(log(n))\)

4. Examples

4.1. 1

for(int i = 1; i<n*n; i+=100) //O(n^2)
    for(int j = n/2; j>1; j--) //O(n)

Here it’s \(O(n^2)\)

But we can say that it’s \(O(n^3)\) or even \(O(n^1000)\) that’s because the alogrithm will never exceed those as well, however, we generally ask for the ’tightest’ big-Oh

4.2. 2

for(int i = 1; i<n; i++) // O(n)
    for(int j =n; j<=1; j=j/2) // O(log(n))

This will be \(O( O(n) * O(log(n)) )\) ← \(O(nlog(n))\)

4.4. 4

Let’s say we have to algorithms, A1 and A2 , A1 is O(n) and A2 is O(n2)

Let’s say A2 takes 1 hour to finish for an input of 100, how much time would it take to finish for a n input of 200?

First of all in 1 hour we can do n2 ← (100)2 operations, and an input of 200 would need (200)2 operations. 1 hour = 10000 operations, 40000 operations = 4 hours!

How much time would it take to finish using A1?

Since we know we can do 10000 operations in hour, for an input of 100, A1 would take \(\frac{100}{10000}\) so \(\frac{1}{100}\) hours.
