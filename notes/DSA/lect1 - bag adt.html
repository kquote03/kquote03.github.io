1. What IS an array??

It is a contiguous fixed strip of memory. We must specify the size of an array.

2. Bags

A simple ’container’ that:

  • Doesn’t store order
  • Allows for repetition

2.1. Bag operations (interface)

Please note that these are not valid method names. Actual method naming depends on the programming language.

2.2. add

Adds an item to the bag, adds it after the last item. It returns ’true’ if the item was indeed added, false if something went amiss.

2.3. retrieve / remove

Retrieves an item from the bag. It always gets the last item, since anyways we don’t keep track of order, and any other way will take more instructions (steps) since we will be leaving a gap in the array.

2.4. retrieve / remove (specific)

Looks for the specified item, and retrieves the first instance it finds of the item. This leaves a gap, the strat to combat this is replacing the gap with the last item.

We do not do this because let’s say we have 10000 items, then that would mean we need to shift 9999 times! The other method just takes 1 instruction.

2.5. get number of items

Retrieves the number of items in the bag

2.6. is bag full

Tells us if the bag is full, we just compare the number of items to the size of the array.

2.7. get frequency

Tells us how many times an item is present in a bag.

We will use the simple way of looping over the whole bag, keeping a counter, and incrementing the counter each time the item matches the current item in the loop.

3. Implementation

We need to know the size of the bag. Since we are implementing bags using an array, we can ask the user to specify the size of the array, or at least set the default size.

Since we must also specify the data type of the array, we can use the ’Object’ type to encompass all other types.

We also need to implement a counter to know the location of the last item. 

BagInterface<String> attendance = new ArrayBag(10);
System.out.println(attendance.isEmpty()); //Should return true at this point
System.out.println(attendance.getCurrentSize()); //Should return 0 at this point
/* The output of the above code is NOT 10,
  10 is the size of the array, NOT the Bag.
*/

attendance.add("Shaima");
attendance.add("Raghad");
attendance.add("Raghad");

System.out.println(attendance.isEmpty()); //Should return false at this point
System.out.println(attendance.getCurrentSize()); //Should return 3 at this point

System.out.println(attendance.remove()); //Should return "Raghad"
System.out.println(attendance.getCurrentSize()); //Should return 2 at this point

for(int i = 0; i<10, i++){
attendance.add("Mohammed");
    } //This will only add 7 "Mohammed" because after the 8th the bag will be full.

System.out.println(attendance.add("Mohanad")); //Returns false because the bag is full now.

System.out.println(attendance.remove("Raghad")); //This version of the overloaded method 'remove'
//returns either true or false if the item was removed.

printBag(attendance); //this will print all the items from bottom to top (Mohammed 8 times, Raghad, Shaima)

attendance.remove("Raghad");
printBag(attendance);
/* This will print Mohammed 7 times, then instead of Raghad, we print another Mohammed, then Shaima
 */

3.1. A function to print all items in the bag whilst destroying the bag

public static void printBag(BagInterface bag){
while(!bag.isEmpty()){
        System.out.println(bag.remove());
    }
}

/*This continuously prints the last item,
  and removes it. It stops until the bag
  is empty.
*/
